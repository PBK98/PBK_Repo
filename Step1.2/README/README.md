1. 프로젝트 개요
    "클래스(Class)를 활용한 파이썬 객관식 퀴즈 게임 만들기"
    사용자 관리: 여러 사용자를 등록하고, 점수를 따로 관리하며, 마지막 접속자를 기억합니다. (UserManager, state.json)
    퀴즈 기능: 퀴즈를 풀고, 새로운 퀴즈를 추가하고, 전체 퀴즈 목록을 보고, 특정 퀴즈를 삭제할 수 있습니다.
    점수 및 힌트 시스템: 정답을 맞히면 점수를 얻고, 힌트를 사용하면 점수가 감점되는 기능이 있습니다.
    데이터 영속성: 퀴즈 내용과 사용자 정보를 JSON 파일에 저장하여, 프로그램을 껐다 켜도 데이터가 그대로 유지됩니다.
    견고한 프로그램 구조: try-except를 사용한 예외 처리.
    객체 지향 설계: UserManager, SolveQuiz 등 여러 클래스를 만들어 각자 역할을 명확하게 분담하는 방식으로 코드를 체계적으로 작성하고 있습니다.

2. 퀴즈 주제와 선정 이유
    Codyssey 에 관한 문제들로 구성했습니다.
    - 해당 과제를 평가하는 동료들이 편하게 풀 수 있는 주제가 뭘까 생각하다가 현재 진행중인 Codyssey에 관해서 쉽게 알 수 있는 문제들이 좋겠다고 생각했습니다.

5. 파일 구조
        QuizGame/
        ├── main.py           # 프로그램 시작 및 메인 메뉴 관리
        ├── QuizGame.py       # 핵심 로직 및 기능 클래스 모음
        ├── Quiz.py           # (현재 사용되지 않음) 초기 퀴즈 클래스
        ├── user.json         # 사용자 정보 데이터베이스
        └── state.json        # 퀴즈 문제 데이터베이스

        
6. 데이터 파일 설명
    main.py (실행 파일)
        역할: 프로그램의 **진입점(Entry Point)**이자 전체 흐름을 제어하는 지휘자입니다.
        주요 기능:
        QuizGame.py에 정의된 모든 기능 클래스들의 인스턴스를 생성합니다.
        사용자에게 메인 메뉴를 무한 루프로 보여주고 입력을 받습니다.
        사용자 선택에 따라 해당 기능 클래스의 메서드를 호출합니다.
        try-except 구문을 통해 프로그램 전체에서 발생할 수 있는 예외를 처리하여 안정성을 확보합니다.
    
    QuizGame.py (핵심 로직)
        역할: 프로그램의 모든 핵심 기능을 담당하는 모듈입니다. 각 기능은 독립된 클래스로 구현되어 있습니다.
        주요 클래스:
        UserManager: 사용자 추가, 이름 변경, 삭제 및 현재 로그인된 사용자(current_user) 상태를 관리합니다.
        SolveQuiz: UserManager와 협력하여 퀴즈 풀이 과정을 진행하고 점수를 계산 및 업데이트합니다.
        AddQuiz, RemoveQuiz, AddHint: 퀴즈 문제에 대한 생성(Create), 삭제(Delete), 수정(Update) 기능을 각각 담당합니다.
        QuizList, ShowScore: 퀴즈 목록과 전체 사용자 점수를 조회(Read)하는 기능을 담당합니다.
        보조 함수:
        load_data(filename): JSON 파일의 데이터를 읽어와 파이썬 딕셔너리로 변환합니다.
        save_data(filename, data): 파이썬 딕셔너리 데이터를 JSON 파일에 저장합니다.
    
    Quiz.py (보조 파일)
        역할: 현재 프로젝트의 메인 로직에서는 사용되지 않습니다.
        내용: 초기 개발 단계에서 사용되었던 퀴즈 클래스로 추정되며, 현재는 모든 기능이 QuizGame.py의 클래스들로 이전되었습니다.
    
    user.json & state.json (데이터 파일)
        
        user.json: 모든 사용자 정보를 저장합니다.
            "사용자이름": {"score": 점수, "solved_count": 푼 횟수} 형태의 딕셔너리 구조를 가집니다.
    
        state.json: 퀴즈 문제와 관련된 모든 데이터를 저장합니다.
            "questions" 키에 문제, 선택지, 정답, 힌트가 포함된 퀴즈 객체 리스트를 저장합니다.
